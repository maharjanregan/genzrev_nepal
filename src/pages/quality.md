This report is meant to be read by humans, not machines.

## Content quality rules

- Prefer **short paragraphs** (mobile-friendly).
- Use **H2/H3 headings** to chunk information.
- Lead each page with **Key findings** (3–7 bullets).
- Avoid vague claims. If we can’t support it, we don’t say it.

## Fact-checking workflow (recursive)

1) **Make the claim** in plain language.
2) Add at least **one credible source link** directly under the claim.
3) Try to find a **second, independent source**. If we can’t:
   - label the confidence as Medium/Low, and
   - explicitly note the gap.
4) If measurement/attribution is uncertain (e.g., partial disruptions), keep the language careful:
   - “measurement indicates…”
   - “reported by…”
   - “attribution unclear…”

## Source hierarchy (preferred)

1) Primary measurement / documentation (e.g., OONI)
2) Rights orgs / UN bodies / press freedom monitors
3) Major international reporting with named sources
4) Local reporting (useful, but cross-check)
5) Opinion columns (context only; avoid using as sole evidence)

## Confidence labels

- **High**: corroborated by multiple strong sources or primary measurement
- **Medium**: credible but limited corroboration / caveats
- **Low**: plausible, but evidence is thin

## What we will not do

- No platform hype.
- No moral framing as substitute for evidence.
- No claims about algorithms/metrics without a source or explicit uncertainty.
